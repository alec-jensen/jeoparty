- ✓ **Schema:** Added `finalCategory`, `finalQuestion`, `finalAnswer` columns to `boards`. Drizzle Kit auto-pushes the migration. `src/lib/schema.ts`, `src/lib/schema.sqlite.ts`
- ✓ **Board editor:** Dedicated "FINAL JEOPARTY" tab at the end of the round tabs. Shows ✓ when ready, ! when missing. Validates on save. `src/pages/boards/[id].astro`, `src/pages/api/boards/[id].ts`
- ✓ **Server flow rewrite** (`src/lib/ws.ts`):
  - `START_FINAL_JEOPARDY` reveals the category to players for wagering. Eligible = score > 0. Players with score ≤ 0 see a "spectating" screen.
  - `FINAL_WAGER` clamped to `[0, currentScore]`.
  - When all eligible wagers are in: broadcast `FINAL_JEOPARDY_START` with the clue → 30s timer on each player's device → auto-submit on expiry.
  - `FINAL_ANSWER` recorded; when all in (or 30s expires) the host gets `FINAL_JEOPARDY_REVEAL` with everyone's answers + correct answer.
  - Host marks each `JUDGE_FINAL` privately (no score change yet).
  - Host clicks `REVEAL_FINAL_RESPONSE` for one player at a time — server then applies the wager to the score and broadcasts `FINAL_RESPONSE_REVEAL` to everyone (presenter shows the dramatic reveal). Host UI suggests revealing incorrect first.
  - After all eligible players revealed → `GAME_OVER`.
- ✓ **Host UI** (`src/pages/game/[id]/host.astro`): Final Jeopardy modal lists each response with WRONG/CORRECT buttons (private judgment) and a REVEAL button that fires once judged. `menuFinal` now sends `START_FINAL_JEOPARDY`.
- ✓ **Presenter UI** (`src/pages/game/[id].astro`): handles `START_FINAL_JEOPARDY` (shows category), `FINAL_JEOPARDY_START` (shows clue + plays `think_music`), `FINAL_RESPONSE_REVEAL` (dramatic per-player reveal with score animation).
- ✓ **Player UI** (`src/pages/game/[id]/play.astro`): score-eligibility gate, wager input with proper min/max, 30s timer that auto-submits.
